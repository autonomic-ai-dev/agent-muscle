# agent-muscle

**Remote actuators and heavy compute for AI agents — massive local compilations, Kubernetes execution, and local LoRA model fine-tuning.**

agent-muscle handles the heavy lifting. When a task requires massive compute resources that exceed a local developer laptop's capability, the orchestrator delegates the execution to this peripheral via the `agent-nerves` event bus.

Rust is the muscle; GPUs and Kubernetes are the weight.

```bash
curl -fsSL https://raw.githubusercontent.com/autonomic-ai-dev/agent-muscle/master/scripts/install.sh | bash -s -- --global
agent-muscle worker start --listen
```

**MCP is live immediately** — it exposes tools that allow the agent to dispatch massive asynchronous batch jobs.

---

## Why agent-muscle?

If an AI Agent is going to replace a Senior Software Engineer, it cannot just write python scripts in a sandbox. It needs to deploy clusters, compile legacy C++, and run massive test suites.

1. **Laptop Constraints:** Running `npm install` on a 500-package monorepo while simultaneously running a local 8B LLM will freeze the developer's laptop.
2. **Infrastructure Requirements:** True integration testing requires spinning up databases and microservices inside Kubernetes.
3. **Model Fine-Tuning:** To continuously improve, the agent needs to train task-specific LoRA adapters based on successful code. You cannot train an LLM on an M1 Air while using it.

**agent-muscle acts as the dedicated, remote executor:**

| Problem | agent-muscle answer |
|---------|-------------------|
| "Compiling C++ blocks my IDE" | **Remote Compilation** — offloads massive build jobs to a dedicated build server instantly. |
| "The agent needs to test a microservice architecture" | **Kubernetes Execution** — dynamically deploys generated code into a local or remote k8s cluster for integration testing. |
| "My local LLM isn't smart enough for this specific repo" | **Local LoRA Fine-Tuning** — dynamically trains tiny, task-specific adapters based on successful workflow outcomes. |

## Architectural Deep Dive

`agent-muscle` is a generic worker node. You can install it on a massive AWS EC2 instance or a local bare-metal GPU rig.

### 1. Actuator Plugins
The muscle is dumb. It only knows how to execute plugins.
- **`plugin-k8s`**: Reads a Helm chart generated by the agent and executes `helm install`.
- **`plugin-lora`**: Wraps MLX or HuggingFace to run a PEFT LoRA fine-tuning job overnight on the agent's memory graph.
- **`plugin-build`**: Handles generic makefiles and cargo builds in isolated containers.

### 2. Asynchronous Polling
Because jobs take hours, `agent-muscle` connects to the `agent-nerves` broker.
- When the job finishes, it publishes a `job.complete` event.
- If the remote server loses power, `agent-muscle` detects the failure on reboot and resynchronizes the state with the cluster.

## Complete Setup (Copy & Paste)

### 1. Install the binary

```bash
curl -fsSL https://raw.githubusercontent.com/autonomic-ai-dev/agent-muscle/master/scripts/install.sh | bash -s -- --global
```

### 2. Configuration (`~/.agent_muscle/config.yaml`)

```yaml
worker:
  node_name: "gpu-rig-01"
  nerves_url: "nats://192.168.1.50:4222"

capabilities:
  gpu: true
  docker: true
  k8s: false
```

### 3. Verify

```bash
agent-muscle version
agent-muscle status
```
